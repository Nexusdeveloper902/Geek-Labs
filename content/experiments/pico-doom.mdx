Can it run DOOM? The eternal question in hardware hacking. In this experiment, I set out to prove that not only can the Raspberry Pi Pico run DOOM, but it can run it well — and on a tiny monochromatic OLED display!

To pull this off, the standard 133 MHz wasn't going to cut it. I pushed the RP2040 chip up to **250 MHz** to ensure a smooth framerate while rendering the game logic and pushing pixels to the screen.

### The Setup & Code

Here is the brains of the operation. Pushing the RP2040 to 250 MHz requires some careful configuration, but unlocking that extra performance is what makes this project possible. I wired up a classic analog joystick for movement and a couple of momentary push buttons for firing.

Getting the DOOM engine to run effectively meant porting the display logic and inputs to the Pico SDK.

It's a beautiful intersection of raw embedded C++ performance and classic game programming. The RP2040's dual-core architecture allows us to handle the game state on one core while managing display updates on the other.
