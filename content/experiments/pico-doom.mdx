Can it run DOOM? That's always the first question when a new microcontroller drops. For this experiment, I wanted to see how far I could push the Raspberry Pi Pico by rendering a DOOM-style first-person shooter on a tiny monochromatic OLED display! 

To be completely fair, this isn't the *full* DOOM engine — it's a wireframe 3D clone with some basic enemies you can run around and shoot. But to pull off those 3D calculations efficiently, the standard 133 MHz wasn't going to cut it. I had to overclock the RP2040 chip up to **250 MHz** to keep things running smoothly.

### The Setup & Code

Getting the code to run effectively meant porting some display logic and mapping out inputs to the Pico SDK. The RP2040's dual-core architecture is incredibly handy here, letting us crunch math on one core while managing display updates on the other.

For the controls, I kept it minimal. I wired up a standard analog joystick to handle the movement, and rather than adding external buttons, I just used the joystick's integrated push-button to fire. Everything connects neatly via I2C to standard SSD1306 OLED screen.

### The Final Result

Seeing the 3D lines render across the tiny display feels really rewarding.
